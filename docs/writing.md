# Documentation Writing Instructions

Standards for writing documentation in this project.

## Style

- Concise, straight to the point
- Non-verbose
- No filler words or pleasantries

## Structure

- Use progressive disclosure - main docs link to detailed sub-docs
- Keep individual files focused on one topic

## Formatting

- Prefer bullet lists over tables (easier to read)
- Use headers to organize sections
- Code blocks for commands and file paths
